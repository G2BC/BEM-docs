Definition of Done

Team:  Pedro Benevides, Rafael Cruz, Uendel Lima, Dêivísson Gomes, Italo Cruz.

Product: Critérios de Conclusão do Sistema Web Fungi: Plataforma de Gerenciamento do Projeto 'Brazilian Edible Mushrooms'.

DoD Estória (Funcionalidades):

Geral:

  • Estórias e Atividades representadas no Trello, especificamente na lista 'PRODUCT BACKLOG'.
  • Código versionado em Branches para cada Atividade no GitHub, seguindo o padrão de nomenclatura em camel case (objeto da atividade/descrição simples atividade, por exemplo: "feature/authUser"), com commits semânticos (por exemplo: "feature(AuthController): implementa rota de autenticação").
  • Testes Unitários validados e armazenados no GitHub.
  • Código revisado por pares através de Pull Requests (PRs).

Adicionais Backend:

  • Documentação da API em Markdown, incluindo Método HTTP, Rota, Objeto de Retorno, Código HTTP e Body (se houver).
  • Diagrama de Classes atualizado em sintaxe Mermaid ou utilizando draw.io, versionado no GitHub no repositório da documentação.
  • Diagrama de Banco de Dados (DER) atualizado em sintaxe Mermaid ou utilizando draw.io, versionado no GitHub no repositório da documentação.

Adicionais Frontend:

  • Layout da Interface feito no Figma, versionado no GitHub no repositório da documentação,para telas com diferentes funcionalidades.
  • Interface Funcional de acordo com o design no Figma.

DoD Sprint:

DoD das Estórias aprovadas.

DoD das Estórias aprovadas, representadas em Cards no Trello na lista 'SPRINT BACKLOG', priorizadas e com tempo previsto e tempo executado.

Relatório de Testes Funcionais em tabela Excel e versionado no GitHub no repositório da documentação, descrevendo o Objetivo do Teste, Especificações de versão, Estórias incluídas no teste e indicação de aprovação ou reprovação.

Documentação do Sistema em PDF e versionado no GitHub no repositório da documentação, abordando como utilizar o sistema, incluindo seções sobre Sobre, Tecnologias Utilizadas, Passo a Passo de instalação e execução (local e docker), Usuários e Estórias.

DoD Release:

DoD da Sprint aprovado, com entrega do código fonte no GitHub em uma branch chamada 'release'.

Release em execução no servidor.
